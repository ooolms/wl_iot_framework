usage:
wliotproxy register_virtual_device <device_id> <device_name> <sensors_xml_file>

<sensors_xml_file> - path to a file containing sensor description in xml format

This command register virtual device with specified id, name and sensors list.
The device is always connected, but by default doesn't process any commands.
Use "wliotproxy vdev_meas" to send values, that are generated by this device.

WARNING: It is not recommended to change device sensors description after adding some storages.
If you change sensor value type, it's storage will not work.
